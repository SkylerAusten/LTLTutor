Let's break down LTL syntax, as used by this tutor:

  1. Atomic Formulae: These are the basic building blocks of LTL formulae. They represent simple properties of the system, like "the light is green" or "the temperature is below 30 degrees." In the grammar, these are represented by lowercase identifiers (e.g., lightgreen).
  2. Logical Operators:
    • Disjunction (|): This operator combines two formulas and evaluates to true if either of them is true. For example, if we have a formula a | b, it means "a or b."
    • Conjunction (&): This combines two formulas and evaluates to true only if both of them are true. For instance, a & b means "a and b."
    • Until (U): This operator represents a temporal relationship between two formulas. It evaluates to true if the first formula remains true until the second formula becomes true. For instance, a U b means "a until b."
    • Implication (->): This operator represents logical implication. It evaluates to false only if the first formula is true and the second formula is false. For example, a -> b means "if a, then b."
    • Equivalence (<->): This operator denotes logical equivalence. It evaluates to true if both formulas have the same truth value. For instance, a <-> b means "a if and only if b."
    • Next (X): This operator represents the next state in the system. It evaluates to true if the formula is true in the next state. For example, X a means "a holds in the next state."
    • Eventually or Finally (F): This operator asserts that a formula will become true at some point in the future. For instance, F a means "eventually a."
    • Globally (G): This operator asserts that a formula is always true from the current state onwards. For example, G a means "a holds from this point on."
    • Negation (!): This operator negates a formula. It evaluates to true if the formula is false and vice versa. For instance, !a means "not a."
    • Parentheses: These are used to group subformulas together and specify the order of operations.

Using this grammar, we can create expressions like G (lightgreen -> (X !red)) to assert that the traffic light is always green, and if it turns red at any point, it will be green in the next state.
